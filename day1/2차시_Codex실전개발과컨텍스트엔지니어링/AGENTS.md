```markdown
# AGENTS.md

## 목표
이 문서는 '랜덤 점심 추천기' 프로젝트의 코드 구조 및 함수 작성 규칙, 커밋 규칙 등을 설명합니다.

## 구조
프로젝트는 다음과 같은 구조로 작성됩니다:

```
/random-lunch-recommender
│
├── index.html        # 메인 HTML 파일
├── styles.css       # 스타일 관련 CSS 파일
└── script.js        # JavaScript 로직 파일
```

- **index.html**: 사용자 인터페이스를 구성하는 HTML 요소를 포함합니다.
- **styles.css**: 웹 페이지의 디자인 및 스타일을 정의합니다.
- **script.js**: 랜덤 점심 추천과 관련된 JavaScript 코드를 포함합니다.

## 한국어 주석 규칙
- 모든 함수 및 주요 코드 블록에는 한국어 주석을 달아 코드의 기능과 목적을 설명합니다.
- 주석은 코드 바로 위에 작성하며, 간결하고 명확하게 표현합니다.
  
예시:
```javascript
// 점심 메뉴를 랜덤으로 추천하는 함수
function recommendLunch() {
    // 메뉴 배열에서 랜덤 메뉴 선택
}
```

## 함수 분리 원칙
- 각 함수는 단일 책임 원칙(SRP)을 따라야 합니다. 즉, 각 함수는 하나의 기능만 수행하도록 작성합니다.
- 함수는 명확한 이름을 가지며, 그 이름만으로도 해당 함수의 역할을 이해할 수 있어야 합니다.

예시:
```javascript
function getRandomItem(array) {
    // 주어진 배열에서 랜덤한 아이템을 반환
}

function displayRecommendation(item) {
    // 추천된 아이템을 화면에 표시
}
```

## 커밋 규칙
- 모든 커밋은 의미 있는 단위로 진행하며, 간단명료한 메시지를 작성합니다.
- 커밋 메시지는 다음과 같은 형식을 따릅니다: `[태그] 메시지`
    - 태그: 기능 추가 (feat), 버그 수정 (fix), 문서 수정 (docs), 스타일 수정 (style) 등
    - 메시지: 해당 커밋이 포함하는 변경 사항에 대한 간단한 설명
  
예시:
```
feat: 랜덤 점심 추천 기능 구현
fix: 버튼 클릭 시 비정상 종료 문제 수정
docs: AGENTS.md 문서 추가
```

## 결론
이러한 규칙을 준수하여 프로젝트를 진행하면 유지보수성과 가독성이 높은 코드를 작성할 수 있습니다. 팀원 간의 협업에 큰 도움이 될 것입니다.
